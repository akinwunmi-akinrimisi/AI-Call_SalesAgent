<task type="auto">
  <name>Task 1: Create Gemini Live API audio round-trip validation script</name>
  <files>scripts/validate_gemini.py</files>
  <action>
Create `scripts/validate_gemini.py` that performs a full Gemini Live API validation with 3 tests:

**Test 1: Connection + Text-to-Audio**
- Authenticate using Vertex AI service account pattern from RESEARCH.md Pattern 1:
  ```python
  from google import genai
  from google.genai import types
  from google.oauth2.service_account import Credentials
  ```
- Load credentials from GOOGLE_APPLICATION_CREDENTIALS env var (default: secrets/openclaw-key-google.json)
- Create client with vertexai=True, project="vision-gridai" (from env), location="europe-west1" (from env)
- Configure LiveConnectConfig with response_modalities=["AUDIO"], voice_name="Aoede"
- Connect to model "gemini-live-2.5-flash-native-audio" (from env)
- Send a text prompt: "Hello, please say 'audio test successful' briefly."
- Receive audio response. PASS if >0 bytes received. Print byte count.
- Save received audio to `scripts/test_output_gemini.pcm` for manual listening test.

**Test 2: Audio-to-Audio (bidirectional)**
- Generate a short PCM 16kHz mono sine wave (440Hz, 1 second) as test input audio
- Send via session.send_realtime_input() with audio=types.Blob(data=pcm_bytes, mime_type="audio/pcm;rate=16000")
- Receive audio response. PASS if >0 bytes received.
- NOTE: Gemini outputs PCM at 24kHz. Print the output size and confirm it is non-zero.

**Test 3: Mulaw transcoding pipeline**
- Import audioop (from audioop-lts package)
- Create synthetic mulaw 8kHz audio (convert the 440Hz sine wave to mulaw)
- Implement and test the two transcoding functions from RESEARCH.md Pattern 3:
  - mulaw_to_pcm16k(mulaw_bytes) -> PCM 16kHz bytes
  - pcm24k_to_mulaw(pcm_24k_bytes) -> mulaw 8kHz bytes
- Test: mulaw input -> pcm16k -> (send to Gemini) -> receive pcm24k -> mulaw output
- PASS if the final mulaw output has >0 bytes
- Save mulaw output to `scripts/test_output_mulaw.raw` for quality assessment
- Print size ratios at each step so transcoding math can be verified

**Error handling:**
- If auth fails, print the exact error and suggest: "Run: gcloud projects add-iam-policy-binding vision-gridai --member=serviceAccount:{sa_email} --role=roles/aiplatform.user"
- If model not found, print available models and suggest checking the model name
- If voice "Aoede" fails, try without specifying a voice and report which voice was used
- Timeout after 30 seconds per test

**Script structure:**
- Use asyncio.run() for the main entry point
- Print [1/3], [2/3], [3/3] progress markers
- Print PASS/FAIL per test
- Print overall PASS (all 3) or PARTIAL (some pass) or FAIL (connection failed)
- Exit code 0 if at least Test 1 passes, exit code 1 if connection fails entirely
