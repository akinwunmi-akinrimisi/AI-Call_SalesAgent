  <action>
    1. Rewrite backend/voice_handler.py (replace stub) with:

       a. fetch_lead(lead_id: str) -> dict | None:
          - Async function. GET from Supabase REST: {supabase_url}/rest/v1/leads?id=eq.{lead_id}&select=*
          - Uses Accept-Profile: sales_agent header (established pattern from Phase 2)
          - Returns lead dict or None if not found
          - Logs "lead_fetched" or "lead_not_found" via log_event

       b. get_firestore_client() -> firestore.AsyncClient:
          - Creates AsyncClient with project=config.gcp_project_id
          - Uses service account credentials from config.google_application_credentials
          - Cache or reuse client across calls (module-level or passed in)

       c. handle_voice_session(websocket: WebSocket, lead_id: str) -> None:
          - Main handler function called from main.py WebSocket route. Full flow:

          SETUP:
          1. Accept WebSocket connection
          2. Fetch lead from Supabase via fetch_lead(lead_id). If not found, send error JSON and close.
          3. Load KB from Firestore via load_knowledge_base(firestore_client)
          4. Build system instruction via build_system_instruction(lead["name"], kb_content)
          5. Create agent via create_sarah_agent(system_instruction)
          6. Create InMemorySessionService and Runner(agent=agent, app_name="cloudboosta-voice-agent", session_service=session_service)
          7. Create session via session_service.create_session(app_name="cloudboosta-voice-agent", user_id=lead_id, session_id=f"call-{lead_id}-{timestamp}")
          8. Create LiveRequestQueue
          9. Create CallSession(lead_id=lead_id, lead_name=lead["name"])
          10. Log "call_started" event

          CONFIGURE RunConfig:
          - streaming_mode=StreamingMode.BIDI
          - response_modalities=["AUDIO"]
          - input_audio_transcription=AudioTranscriptionConfig()
          - output_audio_transcription=AudioTranscriptionConfig()
          - speech_config with Aoede voice, en-GB language
          - If context window compression is available in ADK, enable SlidingWindow

          STREAMING (three concurrent tasks via asyncio.gather):

          upstream_audio():
          - Loop: receive_bytes from websocket
          - Create types.Blob(mime_type="audio/pcm;rate=16000", data=data)
          - Send via live_request_queue.send_realtime(audio_blob)
          - On WebSocketDisconnect: break

          downstream_audio():
          - Loop: async for event in runner.run_live(user_id, session_id, live_request_queue, run_config)
          - If event has audio data in parts: send_bytes to websocket
          - If event.input_transcription.text: call_session.append_user_transcript(text)
          - If event.output_transcription.text: call_session.append_agent_transcript(text)
          - On WebSocketDisconnect: break

          watchdog():
          - Start duration_watchdog(live_request_queue) as the third concurrent task
          - Store task reference in call_session.watchdog_task for cancellation

          CLEANUP (in finally block):
          - Cancel watchdog task if still running
          - Close live_request_queue
          - Extract tool state from ADK session: qualification and call_outcome
          - Populate call_session.outcome from tool state
          - Call process_call_end(call_session)
          - Log "call_ended" event with duration

    2. Update backend/main.py:
       - Import handle_voice_session from voice_handler
       - Add WebSocket route:
         @app.websocket("/ws/voice/{lead_id}")
         async def voice_call(websocket: WebSocket, lead_id: str):
             await handle_voice_session(websocket, lead_id)
       - Remove the TODO comment for Phase 3 agent routes
       - Keep the TODO comment for Phase 6 Twilio routes
       - Add WebSocket import from fastapi

    IMPORTANT NOTES:
    - Audio format for Phase 3 (browser): 16kHz PCM 16-bit mono input, 24kHz PCM mono output. No transcoding needed (browser handles this). Twilio transcoding (mulaw 8kHz) is Phase 6.
    - The session_id should include a timestamp to support multiple calls to the same lead: f"call-{lead_id}-{int(time.time())}"
    - Use return_exceptions=True in asyncio.gather to handle partial failures gracefully
    - The finally block MUST always run (even on exceptions) to prevent resource leaks and ensure outcome is written
