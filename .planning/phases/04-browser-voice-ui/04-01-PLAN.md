    **5. Create frontend/src/audio/pcm-player-processor.js:**
    AudioWorklet processor class `PCMPlayerProcessor`:
    - Constructor: allocate ring buffer (Float32Array, size 24000 * 180 = 4,320,000 for ~3 min), writeIndex=0, readIndex=0
    - port.onmessage handler:
      - If event.data.command === "clearBuffer": set readIndex = writeIndex (barge-in flush), return
      - Otherwise: treat event.data as ArrayBuffer of Int16, convert to Float32 (divide by 32768), write to ring buffer circularly
    - process(inputs, outputs): read from ring buffer into output[0][0], advance readIndex (don't advance past writeIndex)
    - Register as "pcm-player-processor"

    **6. Create frontend/src/utils/pcm.js:**
    Export three functions:
    - `convertFloat32ToPCM16(float32Array)`: Returns ArrayBuffer of Int16 PCM. Clamp each sample to [-1,1], multiply by 0x7FFF.
    - `computeRMS(float32Array)`: Returns RMS amplitude (0-1 range). Square each sample, mean, sqrt.
    - `computePCMAmplitude(arrayBuffer)`: Returns RMS of Int16 PCM ArrayBuffer. Create Int16Array view, normalize each sample by /32768, compute RMS.

    **7. Create frontend/src/hooks/useVoiceSession.js:**
    Custom React hook `useVoiceSession()` that manages the full audio pipeline:

    **State:**
    - `status`: "idle" | "connecting" | "connected" | "ended"
    - `transcripts`: array of {speaker, text, id} objects
    - `userAmplitude`: number (0-1) for visualization
    - `sarahAmplitude`: number (0-1) for visualization
    - `callStartTime`: Date or null

    **Refs:**
    - wsRef (WebSocket), recordContextRef (AudioContext 16kHz), playContextRef (AudioContext 24kHz), recorderNodeRef, playerNodeRef, streamRef, isSarahSpeakingRef

    **Key function: `startCall(leadId)`:**
    1. Set status = "connecting"
    2. Create recording AudioContext with sampleRate: 16000 (MUST be in click handler context for user gesture policy)
    3. Create playback AudioContext with sampleRate: 24000
    4. Load AudioWorklet modules using `new URL("../audio/pcm-recorder-processor.js", import.meta.url)` and similar for player
    5. Request microphone: `navigator.mediaDevices.getUserMedia({ audio: { channelCount: 1 } })`
    6. Wire recording: source -> AudioWorkletNode("pcm-recorder-processor") -> recorder.port.onmessage sends binary to WebSocket
    7. Wire playback: AudioWorkletNode("pcm-player-processor") -> playContext.destination
    8. Open WebSocket: auto-detect URL from `window.location` -- use `ws://` for http, `wss://` for https, path = `/ws/voice/${leadId}`
    9. Set ws.binaryType = "arraybuffer" (CRITICAL -- prevents Blob default)
    10. ws.onopen: set status = "connected", set callStartTime = new Date()
    11. ws.onmessage: if ArrayBuffer -> post to playerNode.port + compute sarahAmplitude + set isSarahSpeakingRef=true; if string -> JSON.parse, add to transcripts
    12. ws.onclose: set status = "ended"
    13. ws.onerror: log error, set status = "ended"

    **Barge-in in recorder onmessage:**
    - After converting Float32 to PCM16 and sending to WebSocket, compute RMS of the Float32 data
    - Update userAmplitude state (throttle to every 3rd frame to reduce renders)
    - If RMS > VAD_THRESHOLD (0.015) AND isSarahSpeakingRef.current is true:
      - Post { command: "clearBuffer" } to playerNode to flush playback
      - Set isSarahSpeakingRef.current = false

    **Sarah amplitude tracking:**
    - On each binary WebSocket message, compute PCM amplitude using computePCMAmplitude
    - If amplitude > 0.005, set isSarahSpeakingRef.current = true; update sarahAmplitude state (throttled)
    - If no audio received for 200ms (use setTimeout), set isSarahSpeakingRef = false, sarahAmplitude = 0

    **Key function: `endCall()`:**
    1. Close WebSocket if open (code 1000)
    2. Stop all MediaStream tracks
    3. Close both AudioContexts
    4. Set status = "ended"

    **Cleanup:** Return cleanup function that calls endCall() on unmount.

    **Return:** { status, transcripts, userAmplitude, sarahAmplitude, callStartTime, startCall, endCall }

    **React Strict Mode guard:** Use a ref flag to prevent double WebSocket creation. Check wsRef.current before opening new connection.

    **VAD_THRESHOLD constant:** Define at top of file as 0.015. Add comment that this can be tuned.
